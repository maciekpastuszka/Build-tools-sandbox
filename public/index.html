Struktura plików

Katalogi/pliki CSS zostały podzielone według struktury ITCSS link.

  • W katalogach 0_settings oraz 1_tools nie powinno być deklaracji klas a jedynie informacje konfiguracyjne, zestaw funkcji, mixinów itp.
  • 2_generic przeznaczony jest na narzędzia do zerowania/normalizowania styli oraz ustawiania globalnego box sizingu.
  • 3_elements to style z podstawowymi elementami strony, które są reużywalne np nagłówki. Konwencja nazewnictwa klas - Atomic link.
  • 4_objects Style dla struktury layoutu np grid, container itp. Domyślnie zastosowany jest custom grid jednak na potrzeby projektu można zastąpić np Bootstrapem.
  • 5_components Style dla elementów strony, to style nad którymi przyjdzie nam pracować najwięcej. Konwencja nazewnictwa klas - BEM link
  • 6_helpers To wszelkiego rodzaju modyfikatory nadpisujące powyższe style. np is-grey. Konwencja nazewnictwa klas - Atomic

Funkcje pomocnicze

  • em($px) - Konwerter px na em. Przykładowo font-size: em(16px); => font-size: 1em; (Przy default font size: 16px)

Mixiny

  • @include breakpoint($width) => @media only screen and (min-width: $width)
  • @include container()
  • @include row($margin) - $margin = wartość ujemnego marginesu. Wartość domyślna przypisana do zmiennej $grid_columns-padding (domyślnie 15px)
  • @include xs($col, $padding) - $col - liczba kolumn, $padding - padding kolumny (domyśla wartość przypisana zmiennej: $grid_columns-padding) odpowiednik np klasy .xs-6 z dodatkowym ustawieniem paddingu
  • @include retina() - Tło obrazkowe dla ekranów z retiną
  • @include section-bg() - Tło obrazkowe RWD
